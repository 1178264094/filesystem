Issue Reporting

Boost.Filesystem issues such as bug reports or feature requests can be reported via a Boost Trac Ticket.

GitHub pull requests are encouraged, too, although for anything beyond really trivial fixes a trac ticket mentioning the pull request should also be submitted.

Bug reports

A quick response to your bug report is much more likely if the problem can be immediately reproduced without any guesswork.

By far the most productive way to do that is to:

  1. Provide a simple test program that automatically yields an unambiguous pass or fail result.
  2. Describe the compiler, standard library, platform, and Boost version you used to build and run your test program.
  3. Describe how a library maintainer can build and run the test program.
  4. Supply a copy or screen shot of the output you get when you run the test program.

Bug reporting boilerplate

The header <boost/detail/lightweight_test_config.hpp> supplies boilerplate code that automatically supplies most of the information requested in (1) and (2).

Example

#include <boost/detail/lightweight_test_config.hpp>

int test_main(int, char*[])
{
  BOOST_TEST(2+2 == 6);
  BOOST_TEST_EQ(2+2, 6);
  return ::boost::report_errors();
}

The output will look something like this:

Microsoft Visual C++ version 14.0
Dinkumware standard library version 610
Win32
Boost version 1.58.0
Command line: ...lightweight_test_report_example.exe
...lightweight_test_report_example.cpp(5):
 test '2 + 2 == 6' failed in function 'int __cdecl test_main(int,char *[])'
...lightweight_test_report_example.cpp(6):
 test '2 + 2 == 6' failed in function 'int __cdecl test_main(int,char *[])': '4' != '6'
2 errors detected.
